Project Overview

Large Language Models can generate fluent responses but lack reliable access to external knowledge. On the otherhand, Retrieval-Augmented Generation (RAG) integrates external knowledge sources to produce grounded answers.

In this project you will design and implement a Personalised Multimodal Chatbot that retrieves information from a custom knowledge base containing multiple modalities (text, images, or other media).

The system must focus on:

  • Multimodal knowledge indexing
  • Retrieval design
  • LangGraph agent orchestration
  • Evaluation of query effectiveness

Note: User interface design is not the focus of this project.

Task Requirements

  • A multimodal chatbot backed by a personal knowledge base (Any data of your choice)
  • LangGraph + LLM (Ollama or API)
  • At least two data modalities (e.g., text + images)
  • Implementation of vector indexing (VectorDB)
  • Evaluation of retrieval performance (Test and Case Studies). Critically compare the performance of the LLM with and without the customised Agent pipeline.
  • At least 4 queries of different types and complexity (e.g., Simple: text query, image query; Complex: analytical query, conversational query, deep-research, advance-tool usage). Each query is worth 1 point. See below detailed rubric for more examples.

Example Chatbot Scenarios

Students may design their chatbot around any domain. The following examples illustrate possible directions.

Research Paper Assistant

Assume you have access to an academic paper database, and you indexes the research papers and figures.

"What papers discuss large language models?"
"Show diagrams related to transformers."

Recipe Knowledge Assistant

Indexes recipes and food images.

"Find recipes containing chicken and mushrooms."
"Identify this dish from an image."

Travel Guide Chatbot

Indexes travel locations and landmark images.

"What attractions are located in Kyoto? Show me some photos."
"What building is shown in this photo?"

Product Knowledge Chatbot

Indexes product catalogues and specifications.

"Find laptops with at least 16GB RAM."
"Identify this product from an image."

Movie & TV Show Navigator

Indexes film plots, cast details, and movie posters.

"Recommend sci-fi movies directed by Nolan."
"What movie is this still from?"

E-commerce Fashion Stylist

Indexes clothing items, outfit inspirations, and style tags.

"Find winter jackets under $100."
"Find some shoes that match this dress, show me how they look."

Detailed Rubric

Knowledge Base

2 Marks
Data Integration 1.0

Collects and integrates a dataset containing at least two distinct data modalities (e.g., text and images) into a structured format. The dataset size should not be too small (e.g., more than 10000 samples).

Preprocessing 1.0

Clearly and thoroughly documents the data preprocessing pipeline.

Multimodal Indexing

4 Marks
Vector Store Base 1.0

Successfully sets up and populates a vector database with your selected dataset.

Embeddings 1.5

Generates and handles multimodal embeddings correctly (e.g., using CLIP for images/text alignment, or combining separate embeddings effectively).

Strategy Comparison 1.5

Compares at least two indexing strategies and justifies the final architectural choice with evidence.

Example Query Tasks

4 Marks

Note: These are just examples. You can perform whatever query is suitable for your domain, each worth 1 point. However, you must demonstrate the usage of your agent and its tools.

Text Query 1

Reliably retrieving relevant context and generating a grounded text response.

Multimodal Query 1

Supporting cross-modal capabilities (e.g., text-to-image search, image-to-text retrieval).

Analytical Synthesis 1

Retrieving context from multiple distinct sources to form a comparative or synthesised answer.

Conversational Memory 1

Allows the agent to correctly retain and reference the prior context or media over sequential follow-up questions.

Agent Pipeline

2 Marks
Workflow Execution 1.0

Implements a functional, stateful workflow using LangGraph to orchestrate the retrieval and generation tools.

Tool Selection 1.0

Implements more than 3 tools for different functionalities related to the chosen task, and the agent correctly selects the best tools for the query.

Report

6 Marks
Introduction 1.0

Clearly outlines the chatbot's domain, target use case, and overall structure.

Methodology 2.0

Provides a detailed explanation of the system architecture, embedding choices, chunking strategies, and the LangGraph workflow.

Results & Evaluation 2.0

Includes clear case studies of successful and failed queries, critically evaluating the retrieval accuracy and response quality. You must identify how a standard LLM (without an agent pipeline) would fail to perform the task.

Clarity 1.0

Presentation is well-structured, with appropriate use of diagrams/tables, and proper citations.

Documentation & Code

2 Marks
Usability 1.0

Provides a clear, step-by-step README file explaining how to set up the environment, seed the database, and run the chatbot.

Code Quality 1.0

Code is modular, easily readable, follows best practices, and includes appropriate comments describing complex logic.
